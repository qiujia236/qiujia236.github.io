Vue 完整版与运行时版

Vue 完整版

在初次使用 vue.js 时，可以看到官方提供了两个版本可以使用，包括 vue.js（完整版）和 vue.runtime.js（运行时版）。

  1. 完整版：同时包含编译器和运行时的版本。
  2. 编译器：用来将模板字符串编译成为 JavaScript 渲染函数的代码。
  3. 运行时：用来创建 Vue 实例、渲染并处理虚拟 DOM 等的代码。基本上就是除去编译器的其它一切。

在实际环境中，应当使用运行时版，这是由于 vue.runtime.js 体积更小，在用户网络情况不好时能更快地加载。

我们知道，vue.js 提供了一套模版，使我们能使用 v-if, v-for 之类的语法来编写 html。但是浏览器并不认识 v-if、v-for 之类的模版，需要通过一个模版编译器将这样的模版转换为浏览器认识的 html。所以 vue.js 的渲染逻辑是将模版字符串转换为 html 字符串，在将 html 字符串插入到页面的 html 中实现渲染。

<body>
  <div id="app"></div>
  <script type="text/javascript" src="./node_modules/vue/dist/vue.js"></script>
  <script type="text/javascript">
    new Vue({
      el: "#app",
      data: {
        msg: "hello指令系统"
      },
      template: `<h4>{{msg}}</h4>>`
    });
  </script>
</body>

Vue 运行时版

vue.runtime.js 和 vue.js 的区别在于，vue.runtime.js 不包含模版编译器来获得整个包体积的减少。显然，如果没有模版编译器，上面代码中的模版字符串就没有办法转化为原始的 html 字符串。但是与其将模版字符串在用户端编译，为何不在开发时本地编译好了再发送给用户呢？这样做就节省了 vue.js 中模版编译器所需要的体积。下面即是使用 vue.runtime.js 实现同样的功能:

<body>
  <div id="app"></div>
  <script
    type="text/javascript"
    src="./node_modules/vue/dist/vue.runtime.js"
  ></script>
  <script type="text/javascript">
    new Vue({
      el: "#app",
      data: {
        msg: "hello指令系统"
      },
      render(h) {
        return h("h4", this.msg);
      }
    });
  </script>
</body>

h 就是 vue.runtime.js 提供的函数，它接收模版字符串中的参数，返回渲染好的原始的 html。

Vue 对象默认提供了一个 render()函数，实现渲染

function() {
  var _vm = this
  var _h = _vm.$createElement
  var _c = _vm._self._c || _h
  return _c("div", { staticClass: "red" }, [_vm._v(" " + _vm._s(_vm.n) + " ")])
}

在实际开发中，由于 h 函数的参数比较复杂，实际开发中通过使用 webpack 的 vue-loader 插件能将 vue 单文件组件（以.vue 结尾的文件）转换为 h 函数所需要的参数。
